# CLAUDE.md

This file provides guidance to Claude Code when working with code in this repository.

## Read this first when starting a session

**Cross-project durable rules.** Read [Marc's claude-rules repo](https://github.com/marc-leonti/claude-rules) at the start of every session in this repo. Two files are the priority:

- `claude_user_rules.md` — the short durable behavioral rules. Required reading. Live version:
  ```
  https://raw.githubusercontent.com/marc-leonti/claude-rules/main/claude_user_rules.md
  ```
- `lessons_learned.md` — the cross-project narrative wisdom log. Read before starting any deep investigation. Live version:
  ```
  https://raw.githubusercontent.com/marc-leonti/claude-rules/main/lessons_learned.md
  ```

Those files apply across all of Marc's projects (the Hat Rule, hedging language, treating pushback as data, the wisdom earned in prior projects) and are repo-committed because the iOS Claude Code sandbox does not persist `~/.claude/CLAUDE.md` between sessions. Treat them as load-bearing. They layer on top of the project-specific rules in this file.

If you want the longer reference form (distilled principles + onboarding for newcomers), also read [`getting_started_with_claude.md`](https://raw.githubusercontent.com/marc-leonti/claude-rules/main/getting_started_with_claude.md) from the same repo.
